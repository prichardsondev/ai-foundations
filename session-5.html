1. Claude.ai overview

This course has mostly used ChatGPT so far. Today adds another major AI assistant to the picture.

Simple framing

Claude is an AI assistant that many people use for writing, analysis, long documents, and project-style work. It solves many of the same kinds of problems as ChatGPT, but it may feel stronger or more comfortable for different tasks.

Classroom rule: Do not marry one model. Learn when to switch tools.

Where Claude often feels strong

  • Clear writing and rewriting
  • Reading and discussing longer documents
  • Analysis across a larger body of context
  • Artifacts and project-style organization

What matters most

  • Use the tool that fits the task
  • Compare output quality, not brand loyalty
  • Keep the same review habits no matter which model you use
  • Save strong prompts so you can test them in more than one tool
Tool Good classroom use What to watch
Claude Writing help, long-context reading, analysis, and organized project-style work Still needs fact-checking, privacy judgment, and human review before sharing important work
ChatGPT Broad general-purpose help, drafting, brainstorming, and hands-on classroom prompting practice Same core limits apply: it can still guess, miss context, or sound more certain than it should

2. n8n + Gmail

This is where AI starts to move beyond a single chat window and into a repeatable workflow.

What workflow automation means

Workflow automation means a system follows a sequence of steps for you. Instead of reading one email and asking AI for help one message at a time, you define a flow: when a certain kind of email arrives, summarize it, pull out the next actions, and prepare the next step.

What n8n is

n8n is a workflow tool that connects services and steps together. In simple terms, it lets you say: when this happens, do these steps next.

It is useful because the flow is visible. You can see the trigger, the AI step, the decision point, and the output.

Why Gmail is a good beginner demo

  • Email is familiar to almost everyone
  • The task is easy to explain
  • Labels make it easier to stay safe
  • You can draft helpful actions without giving the system too much power

Start safe

Label first. Draft second. Delete never, unless you truly understand the workflow.

Safe beginner patterns

  • Start with labeled messages instead of the whole inbox
  • Summarize incoming email before taking any action
  • Extract action items or due dates
  • Create a draft reply instead of sending automatically
  • Keep a person in the loop for anything important

Simple example workflow

  • New labeled email arrives
  • AI creates a short summary
  • AI extracts action items
  • Workflow applies a follow-up label
  • Create a draft reply or send a summary to another tool

What to notice during the demo

The real lesson is not just “email automation.” It is learning how to break a job into a trigger, a useful AI step, a controlled output, and a human review point.

3. Prompts as reusable project assets

This connects the whole course together.

Key idea

Prompts are not one-off magic spells. Good prompts become reusable assets you can layer, improve, and carry from one tool or project to another.

Think in layers

  • System or instruction layer: the core rules for how the AI should behave
  • Personal style layer: your preferred tone, clarity level, and habits
  • Project-specific layer: the exact goal, audience, format, and constraints for one job

Why this helps

  • You do not start from zero every time
  • Your results become more consistent
  • It is easier to compare tools using the same prompt stack
  • Projects become easier to share and improve over time

A simple reusable prompt stack

  • Identity Prompt - who I am, what I do, and what context matters
  • Response Style Prompt - how I want the AI to respond
  • Project Brief Prompt - what this specific project is about

Why this works in class

You can keep the stable parts of a prompt separate from the changing parts. That makes prompts easier to improve, easier to test in more than one AI tool, and easier to reuse on future projects.

Beginner shortcut

If the 3-part structure feels too advanced at first, combine the Identity Prompt and Response Style Prompt into one Foundation Prompt, then pair it with a Project Brief Prompt.

Foundation Prompt = who I am + how I want the AI to respond
Project Brief Prompt = what this specific project is about

Where Halo Lumin fits

Halo Lumin is one practical way to keep prompts, notes, project instructions, and markdown in a clean shareable format. It is useful not only for publishing finished work, but also for maintaining reusable source documents that stay readable and easy to update.

That matters because your prompts and project notes do not have to live only inside one chat. They can live as reusable source documents that you maintain once, share when needed, and reuse across tools. When you want a model to read the source itself, a direct content URL can give it the original document rather than a copy pasted version.

Classroom example

Store a master prompt in Halo Lumin, open the direct content view, and ask Claude or ChatGPT to adopt those instructions before answering. This keeps one prompt source easier to maintain across multiple tools.

4. Edge AI + ESP32 Sense

This is an overview section, not a full device setup lab.

What edge computing means

Edge computing means doing work closer to where the data is created instead of sending everything to a distant cloud service first. In plain English: the device near the real-world event can notice something, decide something, or trigger something locally.

Why edge AI matters

  • It can react faster
  • It may reduce how much data needs to be sent elsewhere
  • It opens the door to lightweight, low-cost physical projects
  • It helps you see AI beyond a browser tab

What an ESP32 Sense board is

At a high level, an ESP32 Sense board is a small development board that can connect sensors and basic inputs to simple local logic. Depending on the setup, you can think about camera input, microphone input, sensors, triggers, local decision making, and sending events to a larger system.

During this session, you may see one board running as a simple example so you can connect the idea to something physical.

Capabilities to think about

  • Camera
  • Microphone
  • Sensors and triggers
  • Local decision making
  • Sending events to a larger system

How to get started

  • Get the board and basic starter hardware
  • Flash known-good starter code
  • Test one thing at a time: camera, sensor, or connectivity
  • Start with one tiny success before adding more features
  • Keep early goals simple and observable

Beginner-friendly project ideas

  • Package watcher
  • Pet detector
  • Smart wildlife camera
  • Room activity notifier
  • Simple smoke or alert concept
  • Sensor-triggered status signal
